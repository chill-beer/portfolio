문제 — 캐싱 없는 useEffect 호출로 중복·무분별 통신, 첫 진입 시 화면 밖 섹션까지 전부 호출. 캐시버스팅(t=)으로 CDN 오프로드가 0이라 모든 요청이 origin에 직격되는 구조.

행동 — React Query(캐싱·중복 제거), IntersectionObserver lazy-load(노출 섹션만 통신), 스켈레톤(레이아웃 안정)을 적용. 섹션 단위로 점진 적용·실기기 검증, 문제 시 즉시 되돌릴 kill-switch로 안정 최우선. 부하 테스트용 유저 시나리오·엔드포인트 가중치 매트릭스도 직접 작성해 RPS 산정 근거를 마련.

결과 — 오퍼월 첫 진입 통신량 약 80% 감소(프론트 단독). 서버 최적화를 병행해 처리 한계가 2~3천 → 5천 RPS(DB 피크 50% 이하)로 늘어, 푸시·이벤트 오픈 직후 동시 진입 스파이크 대응 여력을 확보.
